Links in UNIX (Symlink and Hardlink)

cmd :
ln -> Creates a hard link to a file
Example: ln source.txt hardlink.txt

ln -s -> Creates a symbolic (soft) link to a file
Example: ln -s source.txt symlink.txt

ls -l -> Displays files with link details and permissions
Example: ls -l

rm -> Removes/deletes a file
Example: rm source.txt

unlink -> Removes a link (works like rm for links)
Example: unlink symlink.txt


Types of Links:

1. Soft Link (Symbolic Link):

Similar to a shortcut in Windows

Points to the original file using its path

If original file is deleted, soft link becomes broken (dangling link)

Can work across different file systems

Can link to directories

Has a different inode number than the original file


2. Hard Link:

Direct reference to the same inode as the original file

If original file is deleted, hard link still works

Cannot work across different file systems

Cannot link to directories

Has the same inode number as the original file


Commands with examples:

# Create a sample file first
echo "Hello World" > original.txt

# Create a hard link
ln original.txt hardlink.txt

# Create a soft link (symbolic link)
ln -s original.txt symlink.txt

# View link details (shows inode numbers)
ls -li

# Check content of both links
cat hardlink.txt
cat symlink.txt

# Delete original file
rm original.txt

# After deletion:
# - Hard link still works (cat hardlink.txt)
# - Soft link becomes broken (cat symlink.txt gives error)


Difference between Soft Link and Hard Link:

Feature | Soft Link (Symbolic Link) | Hard Link
--------|---------------------------|----------
Command | ln -s source link | ln source link
Inode | Different from source | Same as source
After deleting source | Becomes broken (dangling) | Still works
Can link directories | Yes | No
Cross file system | Yes | No
File size | Small (stores path) | Same as source
Looks like | Shortcut | Mirror copy


Additional useful commands:

# Find all hard links to a file (same inode number)
ls -li filename

# Find broken symbolic links
find . -type l -! -exec test -e {} \; -print

# Remove a link (does not affect original file)
unlink symlink.txt

# Create symlink to directory
ln -s /home/user/Documents/ mydocs
