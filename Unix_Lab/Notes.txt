Final Revision Notes (Quick Viva Boost)

These are high-frequency viva points + theory traps examiners love to ask:

🔹 /proc Directory (VERY IMPORTANT)
It is a virtual file system created at boot time.
It does NOT exist on disk (stored in memory).
It gets destroyed when system shuts down.
Contains real-time system + process information.

👉 Example:

/proc/meminfo → shows detailed memory usage of system
🔹 free Command (Memory Monitoring)
Shows RAM + swap usage
Default unit = KB

Flags:

-m → MB
-g → GB
-h → human readable (BEST for viva)

👉 Viva tip:

“free command only shows snapshot of memory, not process-level details”

🔹 df Command (Disk Usage)

Used to check file system disk usage

Shows:

Total blocks used
Free blocks available
Mounted file system location

👉 Viva trick:

df = Disk FILESYSTEM usage (not file size)
🔹 du vs df (COMMON QUESTION)
du → file/directory size usage
df → disk partition usage
🔹 vmstat (System Performance)

Shows:

Processes
Memory usage
Paging activity
Block I/O
CPU scheduling

👉 Viva line:

“vmstat gives overall system performance snapshot including CPU, memory and I/O activity”

🔥 Extra Viva Tips (VERY USEFUL)

Always remember:

man <command> gives full manual of any command

Pipes (|) are used to:

Pass output of one command as input to another

grep is for:

Pattern searching in files

sed is for:

Stream editing (replace/delete text without opening file)

awk is for:

Column-wise text processing
